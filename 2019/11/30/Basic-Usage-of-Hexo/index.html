Insert Images

There are multiple ways that can insert images in Markdown, such as using absolute path, relative path, or url path. In Hexo, we can simply put the images we want to inert in source/images and insert it using the following command. This method is easy to use. However, if we have huge amounts of images in our blog, it would be difficult to efficiently manage them. Luckily, Hexo provides a more organized way to many images on a post-per-post basis. We first turn on this convenient approach by setting the post_assert_folder setting in **_config.yml** to true. With asset folder management enabled, Hexo will create a folder every time you make a new post with the **hexo new [layout] <p>** command. This asset folder will have the same name as the markdown file associated with the post. Place all assets related to your post into the associated folder, and you will be able to reference them using a relative path, making for an easier and more convenient workflow.
